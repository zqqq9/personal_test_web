SPA Routing Configuration

For a single-page application to work correctly on Netlify, you need:

  1. A _redirects file in the publish directory with:
/* /index.html 200

This tells Netlify to serve the index.html file for any route, allowing your SPA router to handle the routing.

Common Issues

  • Missing _redirects file in the published directory
  • Incorrect base path in vite.config.ts (should be '/')
  • Relative paths to assets (use absolute paths starting with /)
  • Missing base tag when needed

Check Current Configuration

Try accessing these paths:

  • /netlify.txt - Should display a text file
  • /netlify.html - This page
  • /nonexistent-path - Should load your SPA, not a 404 page
